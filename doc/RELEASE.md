# kiku 배포(출시) 체크리스트 — Google Play

> 플레이스토어 출시 시 등록/선언해야 하는 것들. 아직 출시 단계는 아니고, **미리 정리해두는 참고 문서**.
> 키쿠 MVP는 로그인·서버·결제가 없어 대부분 항목이 단순하다.

---

## 0. 지금 상태 요약
- **로그인·서버·수집 없음** → 데이터 보안/개인정보 선언이 아주 간단.
- 실제로 신경 쓸 핵심은 **① 릴리스 서명 키**, **② 개발자 계정 + 폼 작성** 두 가지.

---

## 1. 앱 서명 (릴리스) — ★가장 중요
개발 중 쓰는 **디버그 키**가 아니라, 출시용 **릴리스 키스토어**로 서명해야 한다.

- **릴리스 키스토어(`.jks`) 생성** — 본인이 직접(비밀번호 포함):
  ```
  keytool -genkeypair -v -keystore kiku-release.jks -keyalg RSA -keysize 2048 -validity 10000 -alias kiku
  ```
- **`build.gradle.kts`에 signingConfig** 추가 → release 빌드에 서명 적용.
- 비밀번호/키 경로는 **git에 올리지 않는다**(`keystore.properties` + `.gitignore`, 이미 `*.jks`/`*.keystore` 무시 중).
- **Play App Signing**: 구글이 최종 서명 키를 대신 보관. 업로드 키로 서명해 올리면 구글이 재서명(현재 사실상 필수).
- ⚠️ **키(또는 업로드 키)를 잃으면 그 앱을 다시 업데이트 못 한다 → 반드시 안전 백업.**

## 2. 개발자 계정 + 본인 인증
- Google Play 개발자 등록 **$25(1회)**.
- **신원 확인** 필수(개인: 신분증/주소/전화, 사업자: 사업자 정보). 확인에 시간 걸릴 수 있으니 미리.

## 3. 데이터 보안 (Data Safety) 선언
- 앱이 수집/공유하는 데이터를 Play Console 양식에 선언(필수).
- 현재 키쿠: **"데이터 수집 안 함"** 으로 간단 통과.
- 나중에 **계정·광고(AdMob)** 붙이면 → 기기ID/광고ID 수집을 반드시 갱신 선언.

## 4. 개인정보처리방침(Privacy Policy)
- URL 필요(블로그/노션/깃허브 페이지 등에 게시 가능).
- 수집 없어도 대개 요구됨. **AdMob 붙이면 필수**(광고가 광고ID 수집).

## 5. 콘텐츠 등급 / 타겟 연령
- 설문 작성 → 등급 자동 부여. 학습 앱이라 전체이용가 예상.
- 타겟 연령대 선언(어린이 대상 아님).

## 6. 민감 권한 선언
- **Foreground Service `mediaPlayback`** 사용 → Play가 사용 사유(백그라운드 오디오 재생) 설명/데모를 요구할 수 있음. 미디어 재생 앱은 대개 통과.
- `POST_NOTIFICATIONS` 등도 심사 대상.

## 7. 스토어 등록 자료
- 앱 아이콘(512), 피처 그래픽(1024×500), 스크린샷(폰), 짧은/긴 설명, 카테고리(교육).

---

## 출시 직전 순서(권장)
1. 릴리스 키스토어 생성 + 서명 설정(§1)
2. `versionCode`/`versionName` 정리, `minifyEnabled` 등 릴리스 빌드 점검
3. AAB(App Bundle) 빌드: `./gradlew :app:bundleRelease`
4. 개발자 계정/본인 인증(§2) — 시간 걸리니 먼저
5. 데이터보안·개인정보·등급 폼(§3~5)
6. 내부 테스트 트랙 → 검토 → 프로덕션

> 준비되면 §1(키스토어+서명 설정)부터 도와줄 수 있음. 키 생성 명령은 비밀번호가 들어가므로 **본인이 직접 실행**.
