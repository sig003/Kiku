# kiku — JLPT 청해 훈련 앱

## 프로젝트 개요

JLPT N4 수험자를 위한 일본어 청해 훈련 안드로이드 앱.
"내가 원하는 문장만, 내가 원하는 방식으로 반복 청취"를 목표로 한다.

- 플랫폼: Android (Kotlin + Jetpack Compose)
- 사용자: JLPT N4 준비 중인 학습자 (1차 사용자는 개발자 본인)
- 출시 전략: Android 우선 → 검증 후 iOS 확장

## 배경 / 동기

유튜브에서 "일본어 100문장(일본어 3회 + 한국어 1회) 듣기" 콘텐츠로 청해 공부 중.
다음과 같은 한계가 있어 직접 앱으로 만들기로 함.

- 문장 난이도가 내 수준과 맞지 않음 (너무 쉽거나 너무 어려움)
- 이미 아는 문장도 계속 반복됨
- 구간 반복 / 모르는 문장만 다시 듣기 불가
- 카테고리별 학습 불가

## 핵심 컨셉

사용자는 "클립"을 듣는다. 각 클립은 여러 문장으로 구성.

각 문장 흐름 예시:

```
일본어 음성 (3회)
↓ 3초 정지 (해석할 시간)
↓ 한국어 해석
↓ 중요 단어 (일본어 → 한국어)
↓ 다음 문장
```

문장별로 별도 구간을 두어 **건너뛰기 / 다시 듣기** 가능.

**듣기 전용 앱이다.** 화면을 보지 않고 라디오/팟캐스트처럼 듣는 것이 기본 사용 방식이라:
- 화면이 꺼지거나 앱이 백그라운드여도 **소리가 계속 나야 한다**(백그라운드 재생 필수).
- 한국어 해석·단어 뜻도 **음성으로 읽어준다**(일본어+한국어 TTS 둘 다 필요).
- 화면 UI는 후순위로 미루고, 1차 조작은 미디어 알림/이어폰 버튼으로 한다.

## 차별화 포인트

"일본어 문장 들려주는 앱"은 이미 많지만, **JLPT 초보자가 직접 만든 청해 훈련 앱**이라는 점이 차별점.
경쟁력은 AI 성우 품질이 아니라 **실제 수험생 입장에서 만든 UX**.

### 카테고리 예시
- N4 생활회화 (駅はどこですか？など)
- N4 회사생활 (会議は何時からですか？など)
- N4 여행 (パスポートを見せてください など)

### 기능 후보
- 난이도 분리 (N5 / N4 / N3)
- 쉐도잉 모드 (일본어 → 3초 정지 → 따라 말하기)
- 모르는 문장 ★ 저장 (나만의 오답노트)
- 속도 조절 (0.7x / 0.8x / 1.0x / 1.2x)

## 기술 스택

### MVP 단계
- Android Studio + Kotlin + Jetpack Compose
- 데이터: JSON 파일 (앱 내장)
- 음성: Android 내장 `TextToSpeech` API
- 서버/DB: **없음**

### 향후 확장
- 서버: Spring Boot (Kotlin)
- DB: Postgres 또는 Supabase
- 스토리지: S3 (mp3 파일)
- AI 성우: ElevenLabs (일본어 자연스러움 우수)
- 문장 생성: GPT (반자동)

## 데이터 구조

```json
[
  {
    "id": 1,
    "jp": "昨日は会社を休みました。",
    "kr": "어제는 회사를 쉬었습니다.",
    "words": ["昨日", "会社", "休む"],
    "pause": 3000
  }
]
```

## 음성 재생 방식

### 방식 1. Android 내장 TTS (MVP 채택)
```kotlin
tts.language = Locale.JAPANESE
tts.speak(currentSentence.jp, TextToSpeech.QUEUE_FLUSH, null, "sentence")
```
- 비용 0원, 인터넷 불필요, 즉시 구현 가능
- 단점: 기기마다 음성 품질 차이

### 방식 2. mp3 앱 내장
- 인터넷 불필요, 구현 쉬움
- 단점: 1000문장 이상이면 앱 용량 부담

### 방식 3. S3 + mp3 (사전 생성)
- ElevenLabs로 mp3 미리 생성 → S3 업로드 → 앱에서 재생
- 실시간 TTS가 아니므로 사용자당 호출 비용 없음
- 1000문장(약 2.8시간) 기준 ElevenLabs 비용 수만원 수준

## 단계별 로드맵

### 1주차 — 환경 셋업
- Android Studio 설치
- Compose 학습
- TTS API로 일본어 한 문장 읽어보기 (기술적 가능성 검증)

### 2주차 — 데이터
- GPT로 N4 문장 100개 생성 (일본어 + 한국어 + 단어)
- JSON으로 정리

### 3주차 — 재생
- Android TTS로 문장 재생
- 일본어 → 정지 → 한국어 → 단어 순서 구현
- 구간 반복 / 건너뛰기

### 4주차 — 출시
- 광고 붙이기
- 블로그 홍보
- 사용자 반응 확인

## 비즈니스 모델

"대박 사업"보다 **"내가 쓰려고 만들었는데 같은 문제 가진 사람도 쓰는"** 모델.

- 초기: 광고
- 중기: 사용자 반응 좋으면 ElevenLabs 적용 (품질 업)
- 후기: 구독 시스템 + iOS 출시
- 홍보 채널: 운영 중인 일본어 블로그 (네이버 + note)
