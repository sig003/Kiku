## 1. 릴리스 서명 (핵심)

### 1.1 개념
Play에 올리는 앱은 **디버그 키가 아니라 출시용 릴리스 키**로 서명해야 한다. 안드로이드는 "같은 키 = 같은 개발자"로 업데이트를 통제하므로, 이 키는 **앱의 인감도장**이다.

### 1.2 한 일
- `keytool`로 **릴리스 키스토어(`kiku-release.jks`)** 생성. 비밀번호는 **본인이 직접 정함**(채팅/문서에 안 남김).
- 비번은 `keystore.properties`(git 제외)로 분리, `build.gradle.kts`가 읽어 `signingConfigs.release`로 서명.
- **Play App Signing**: 첫 업로드 시 구글이 최종 서명 키를 대신 보관 → 우리 `.jks`는 "업로드 키". (분실 시 재설정 여지)
- ⚠️ `.jks`·`keystore.properties`는 **절대 커밋 금지**(`.gitignore`). 키+비번 별도 백업 필수.

### 1.3 검증
- `apksigner`로 릴리스 APK 인증서 확인 → `CN=mhkim`(본인 키)로 서명됨.

## 2. R8 (코드 축소·난독화)

- `isMinifyEnabled` + `isShrinkResources` 활성화 → AAB 용량↓·리버싱 난이도↑.
- **주의**: `kotlinx.serialization` 모델이 R8에 지워지면 JSON 파싱이 깨진다 → `proguard-rules.pro`에 keep 규칙 추가.
- **실기기 검증**: 릴리스 APK 설치 후 클립 6개 정상 로딩·재생 확인(파싱 에러/크래시 없음). keep 규칙 통과.
- 결과물: `app/build/outputs/bundle/release/app-release.aab` (약 2.5MB).

## 5. 내부 테스트 출시

- **내부 테스트 트랙**에 AAB 업로드 → 출시.
- 내부 테스트는 **초대 메일이 안 옴** — "테스터" 탭의 **참여 링크(opt-in URL)** 를 폰에서 열어 가입 → Play에서 설치.
- ⚠️ 기존 sideload 앱(다른 서명)은 **삭제 후** Play 버전 설치(서명 충돌).
- **실기기에 Play로 설치·실행 확인 완료.**
