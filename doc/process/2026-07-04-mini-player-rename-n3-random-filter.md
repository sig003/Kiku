## 1. 재생 화면 UX 4종

실기기 사용 중 나온 피드백 4개를 순서대로 고쳤다.

### 1.1 뒤로가기 버튼 확대
- 상단 뒤로가기(`‹`) 버튼이 38dp로 작아 누르기 힘들었다 → **46dp + 굵은 글리프**로.
- 권장 터치 타깃(약 48dp)에 맞춤. `PlayerScreen.kt`.

### 1.2 컨트롤 고정 + 텍스트 스크롤 (구조 개선)
**문제:** 문장마다 길이가 달라, 재생 버튼이 위아래로 흔들렸다.
**원인:** 화면 전체(디스크·문장·컨트롤 전부)가 **하나의 스크롤 기둥(Column)** 안에 있어서, 문장이 길어지면 아래 컨트롤까지 밀렸다.

**개념 — Compose의 `weight`:** 세로로 쌓을 때, 가운데 영역에 `weight(1f)`를 주면 "남는 높이를 그 영역이 다 차지"한다. 나머지(상단바·디스크·컨트롤)는 자기 크기만 차지하고 **고정**된다.

**해결:** 화면을 이렇게 나눴다.
```
[상단 바]         ← 고정
[디스크]          ← 고정
[문장·해석·어휘]   ← weight(1f) + 내부 스크롤 (길이 변화를 여기서 흡수)
[스크러버·컨트롤·속도] ← 고정(하단 고정)
```
이제 문장이 아무리 길어도 텍스트 영역만 스크롤될 뿐, **재생 버튼은 절대 안 움직인다.**

### 1.3 미니 플레이어 (가장 큰 작업)
**문제:** 재생 중 뒤로 가면(목록으로) 소리는 계속 나는데 **재생 화면이 사라져** 조작할 수 없었다.
**원인:** 재생은 `PlaybackService`(백그라운드 서비스)가 소유해서 화면과 무관하게 계속되는데, 목록 화면엔 그걸 보여줄 UI가 없었다.

**개념 — 서비스 바인딩 위치 올리기:** 화면이 서비스에 "연결(바인딩)"해야 재생 상태를 볼 수 있다. 기존엔 재생 화면만 연결했다. 이번엔 **앱 최상단(`KikuRoot`)에서 연결**해, 목록이든 재생이든 **같은 상태를 공유**하게 했다.

**해결:** 목록 하단에 **미니 플레이어 바**(스포티파이식)를 띄운다.
- 현재 문장 + 재생/일시정지 버튼, 탭하면 재생 화면으로 **복귀**.
- 복귀 시 **재시작하지 않게** 하는 장치(`fresh` 플래그): 새로 여는 경우만 다시 적재하고, 미니바에서 돌아온 경우는 진행 중인 세션을 그대로 둔다.
- 목록 마지막 항목이 바에 가리지 않게 하단 여백(`bottomPadding`)을 준다.

파일: `MainActivity.kt`(신규 `KikuRoot`+`MiniPlayer`), `PlayerScreen.kt`(`fresh`), `ClipListScreen.kt`(`bottomPadding`).

### 1.4 동심원 이스터에그
- 재생 화면 가운데 디스크의 **동심원이 재생 진행도에 따라 작게→크게** 커진다. 뒤로 가면(이전 문장) 자동으로 작아진다.
- **개념 — `animateFloatAsState`:** 값이 바뀔 때 뚝 바뀌지 않고 부드럽게 이어 주는 애니메이션. TTS는 초 단위 위치가 없어 진행도가 "문장 단위(계단식)"라, 이 애니메이션으로 계단을 부드럽게 덮었다.

## 2. 이름 KIKU + 미니 플레이어 닫기

### 2.1 앱/아이콘 이름 → KIKU
- `res/values/strings.xml`의 `app_name`을 `Kiku`→`KIKU`로. 매니페스트가 이 값을 **앱 이름과 런처 아이콘 라벨 둘 다**에 쓰므로 한 번에 바뀐다.
- 앱 안 상단 브랜드 로고 텍스트(`ClipListScreen`의 `BrandBar`)도 `KIKU`로 통일.

### 2.2 미니 플레이어 닫기(✕)
**문제:** 미니바를 닫을 방법이 없었다.
**원인:** 기존 정지(`stopPlayback`)는 소리만 멈추고 **세션 상태(어떤 클립인지)는 남겨** 미니바가 그대로 떠 있었다.
**해결:** 상태까지 비우는 `dismiss()`를 추가하고 미니바에 ✕ 버튼을 달았다.
- `TtsSequencer.clear()`(상태를 기본값으로) + `PlaybackService.dismiss()`(정지 후 clear).
- **진행 위치 저장값은 유지** → 나중에 그 클립을 다시 열면 이어듣기 된다.

## 3. N3 콘텐츠 300문장 + 랜덤 레벨 필터

### 3.1 "단문도 N4와 N3가 다른가?" — 다르다
한 문장이라도 **문법·어휘·구조**로 레벨이 갈린다. 예)
- N4: 明日は雨が降る**かもしれません**。
- N3: 天気予報**によると**、明日は雨が降る**そうです**。

N3는 `～によると/ようになる/ば～ほど/わりに/せいで/おかげで/につれて/ざるを得ない`, 수동·사역수동·피해수동(`叱られる·待たされる·雨に降られる`), 경어(`おっしゃる·いたす`) 등이 들어간다.

### 3.2 N3 300문장 추가
- `assets/clips/n3_office.json`(id11) / `n3_travel.json`(id12) / `n3_daily.json`(id13), 각 **100문장**, `level: "N3"`.
- 카테고리는 `N3 회사생활/여행/생활회화` — `clipArt`(아트)와 `displayCategory`(레벨 접두 제거)가 그대로 붙는다.
- 총 콘텐츠: **N4 300 + N3 300 = 600문장.**

### 3.3 전체 랜덤이 레벨 필터를 따르게
**문제/질문:** N3를 넣으면 "전체 랜덤"이 N4·N3를 섞어 버린다.
**결정:** 히어로 "랜덤"이 **상단 필터(전체/N4/N3)를 따르게** 한다.
- `randomClip(count, level)`에 레벨 필터를 넣고, 그 레벨을 UI→서비스로 전달(`ClipListScreen → MainActivity → PlayerScreen → PlaybackService → randomClip`).
- 히어로 카드 문구도 레벨 반영: `전체 랜덤` / `N4 랜덤` / `N3 랜덤`.
- 재생 화면 🔀로 다시 섞어도 선택 레벨을 유지(`currentRandomLevel`).
