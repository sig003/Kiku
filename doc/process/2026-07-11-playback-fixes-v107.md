# 2026-07-11 — 재생 품질 수정 5종 + 딩동 교체 (v1.0.7)

> 작업 일지. 실기기 사용 중 나온 문제들을 고치고 v1.0.7로 릴리스.

## 0. 한눈에 보기

실기기로 쓰며 나온 문제 5가지를 고쳤다. 대부분 원인이 명확했고, 하나(랜덤 정리)로 여러 증상이 동시에 풀렸다.

## 1. 고친 것

### 1.1 단어 뜻에서 "피해수동" 같은 문법 라벨이 읽힘
- **원인:** 단어 뜻 kr에 `"…먹힘을 당하다(피해수동)"`처럼 문법 라벨이 괄호로 들어가 있고, TTS가 괄호까지 읽음.
- **해결:** `TtsSequencer.vocalize`에서 **문법 용어가 든 괄호만** 제거(`GRAMMAR_LABEL` 정규식: 수동/사역/겸양/존경/전문 등). 화면 자막엔 유지.
- **주의(회귀 방지):** 처음엔 괄호를 전부 지웠다가 퀴즈의 `"(정답)"`까지 사라지는 회귀 발생 → 문법 용어가 포함된 괄호만 지우도록 좁힘. `"(정답)"·"(남에게)"·"(사진을)"`처럼 뜻에 필요한 괄호는 그대로 읽힌다.

### 1.2 대화(DIALOGUE) 랜덤 재생 시 한 목소리로만 나옴 + 맥락 깨짐
- **원인:** `randomClip`이 대화·퀴즈 문장까지 섞어 뽑았다. 대화는 문장 단위로 섞이면 화자(A/B) 매핑이 깨지고, 한 줄만 뽑히면 맥락이 사라진다("급행이라 통과해 버려요"가 홀로 나와 엉뚱하게 들림 — 번역 자체는 정확).
- **해결:** `randomClip` pool을 **`mode == DRILL`(단문)만** 대상으로. 대화/퀴즈는 랜덤에서 제외. → 목소리 문제·맥락 깨짐·"반복 느낌"이 한 번에 해결.

### 1.3 랜덤이 "맨날 같은 것만" 느낌
- 원인은 1.2와 동일(대화/퀴즈가 섞여 지저분). pool을 순수 단문으로 정리하니 개선. (`shuffled()`라 매번 새로 섞이는 것은 정상.)

### 1.4 한국어 물음표 억양
- 데이터는 정상(700문장 중 물음표 100개, 일↔한 불일치 0). **안드로이드 기본 TTS가 한국어 물음표 억양을 거의 안 살리는 엔진 한계.** 사용자 결정으로 **그대로 두기**(어미가 이미 의문형이라 뜻 전달은 됨).

### 1.5 딩동 효과음 교체 + 길이 단축
- 새 소리 `dingdong.mp3`(5.5초) → 실제 소리는 앞 ~1.2초뿐이고 뒤는 여운/무음.
- **1.4초 mono wav로 잘라** `res/raw/dingdong.wav`(124KB, 끝 60ms 페이드아웃). 기존 `chime.wav`·`dingdong.mp3` 삭제.
- 맥에 mp3 인코더(ffmpeg) 없어 `afconvert`+파이썬 `wave`로 처리. 효과음은 wav가 저지연이라 SoundPool에 더 적합.
- 코드: `R.raw.chime`→`R.raw.dingdong`, 대기 950→1400ms.
