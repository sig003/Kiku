## 0. 한눈에 보기

디자이너(Claude Design)가 준 **하이파이 디자인 핸드오프**(다크+골드, 아이팟 감성)를 받아 앱 UI를 싹 갈아엎었다. 기능은 그대로, 껍데기만 교체.

- **라이브러리**(클립 목록): 브랜드바 + "전체 랜덤" 히어로 + 컬렉션 행 + 레벨 필터

## 1. 개념 — 디자인 토큰과 테마

### 1.1 디자인 토큰
디자인에서 확정한 **색·간격·둥글기 값의 이름표**. 예: 배경 `#16181F`, 골드 `#F3C14E`. 화면마다 하드코딩하지 않고 한 곳(`Design.kt`의 `KikuColors`)에 모아 재사용 → 일관성 + 나중에 한 번에 수정 가능.

### 1.2 다크 고정 테마
Compose의 `MaterialTheme`에 색 스킴을 넣는다. 기존엔 시스템 라이트/다크·다이내믹 컬러를 따랐지만, 키쿠는 **브랜드 다크+골드 고정**이라 시스템 설정 무시하고 항상 같은 다크 테마(`Theme.kt`).

### 1.3 왜 HTML을 그대로 안 쓰나
핸드오프의 `ui-redesign.html`은 **레퍼런스(목업)** 일 뿐. 실제 앱은 Compose라, 같은 스펙(색/크기/간격)을 Compose 코드로 **재현**한다. HTML/CSS를 앱에 이식하는 게 아니다.

## 2. 라이브러리 화면 (`ClipListScreen`)

- **브랜드바**: 골드 라운드 사각 안에 `聞` + `Kiku` / `JLPT LISTENING`.
- **"전체 랜덤" 히어로 카드**: 그라데이션 배경 + 골드 보더 + `▶ 100문장 시작` 골드 필 버튼. 우하단 셔플 아이콘 장식.
- **레벨 필터**: `전체 / N4 / N3` 세그먼트. 클립을 `level`로 걸러 보여줌. N3은 콘텐츠 없어 "준비 중" 안내.
- **컬렉션 행**: 한자 아트 타일(`会社`/`旅行`/`会話` + 카테고리별 그라데이션) + `N4 · 회사생활` 라벨 + 제목 + `100문장 · 드릴` + 재생 아이콘.

> **화면 전환**: 목적지가 목록↔플레이어 둘뿐이라 Navigation 라이브러리 없이 `MainActivity`의 상태(`openClipId`)로 전환(무의존).

## 3. 나우플레잉 화면 (`PlayerScreen`)

- **디스크 아트**: 190dp 원 + 동심원 링 3개(Canvas로 그림) + 중앙 `聞`. 우상단 `n/총` 뱃지.
  - *뱃지 잘림 버그*: 원형 클립 안에 두면 모서리가 잘려서, **클립 밖 바깥 Box**에 뱃지를 배치해 해결.
- **문장**: 일본어(읽는 중 골드 강조) / 한국어(토글) / `한국어 숨기기` / 어휘 칩.
- **스크러버**: TTS는 초 단위 위치(seek)가 없어, **문장 진행도(n/총)** 기반 골드 바로 대체.
- **컨트롤(뮤직플레이어 배열)**: `🔀 셔플 · ⏮ 이전 · ▶ 재생(골드 대형) · ⏭ 다음 · ↻ 다시`.
- **속도 세그먼트**: `0.8 / 1.0 / 1.2`(기본 1.0). 중앙 정렬해 **1.0이 재생 버튼과 같은 중앙선**에 오게, `속도` 라벨은 아래로.

## 4. 실사용 피드백 반영 (리디자인 후 다듬기)

실기기로 보며 나온 요청들:

| 요청 | 처리 |
|---|---|
| 기본 속도 1.0으로 | 세그 `0.8/1.0/1.2`, 기본 1.0. "빠름"은 **JP 회차 간격 1.5→2.2초**로 해결 |
| 속도 버튼이 스크롤해야 보임 | 디스크·간격 축소해 **한 화면에** 다 들어오게 |

| 아트 타일 N4 오버레이 지저분 | 타일에서 빼고 **`N4 · 회사생활`** 라벨로 |
| 전체/N4/N3 선택 | **레벨 필터** 추가 |
| 랜덤 버튼 역할 불명확(누를 때마다 바뀜) | 명확한 **토글**로: 골드=무작위 순서, 뮤트=원래 순서(`setShuffle`) |
| 셔플 아이콘 촌스러움(🔀 이모지) | 제공받은 **Lucide 셔플 SVG** → 벡터 드로어블 `ic_shuffle.xml`, tint로 색 |

| kiku → Kiku | 브랜드바 대문자 |
| 아이콘이 앱보다 더 까맘 | 원인은 **런처 캐시**(옛 `#0F1115`). 값은 이미 동일(`#16181F`)·전경 PNG 투명 확인. 완전 삭제 후 재설치로 갱신 |

## 5. 트러블슈팅 메모

- **디스크 뱃지 잘림**: 원형 `clip` 안의 모서리 요소는 잘린다 → 클립 밖 컨테이너에 배치.
- **아트 매칭 버그**: "N4 회사생활"이 "생활"을 포함해 `会話` 브랜치에 잘못 걸림 → **"회사" 먼저 검사**하도록 순서 수정.
- **아이콘이 더 까매 보임**: 코드는 정상. 삼성 런처가 **아이콘 비트맵을 캐시**해서 옛 색을 보여준 것. `adb uninstall` 후 재설치(또는 런처 재시작/재부팅)로 해결. 전경 PNG 모서리 픽셀 = `RGBA(0,0,0,0)` 투명 확인.
- **`Clip.level` 미정의**: 설계 문서엔 넣었지만 모델엔 빠져 컴파일 실패 → `Playback.kt`에 `level` 필드 추가.
