# 2026-07-04 — 목소리가 전부 여자로만 나오던 문제 (문장마다 남·여 번갈아)

> 작업 일지. **안드로이드 개발이 처음인 사람**도 따라올 수 있게, 무엇을/왜/어떻게 했는지와 개념 설명을 함께 남긴다.

---

## 0. 한눈에 보기

**증상:** 재생하면 일본어가 전부 같은 (여성) 목소리로만 나왔다.
**원인:** 앱이 "어떤 목소리로 읽을지"를 지정하지 않아, 기기의 **기본 목소리 1개**만 계속 썼다. 기본값이 여성이라 전부 여자로 들렸다.
**해결:** 언어별로 목소리 2개를 골라, **문장 인덱스 홀짝으로 번갈아** 읽게 했다 → 남/여가 교대로 들린다.

---

## 1. 개념 — TextToSpeech의 "언어"와 "목소리"는 다르다

- `tts.language = Locale.JAPANESE` → "일본어로 읽어라"까지만 정한다.
- 한 언어 안에도 **여러 목소리(Voice)** 가 있다. 어떤 목소리로 읽을지는 `tts.voice = ...` 로 따로 지정해야 한다.
- 지정을 안 하면 그 언어의 **디폴트 목소리 1개**만 쓴다 → 이번 증상의 원인.

이 갤럭시 S24+에는 (이전 조사에서) 일본어 목소리가 4종 있었다: `htm / jab / jac / jad`. 있어도 **안 골랐으니** 안 쓰였던 것.

## 2. 무엇을 했나 — `TtsSequencer`에 목소리 번갈아

`app/.../TtsSequencer.kt`에 아래를 추가:

1. **목소리 2개 선정 (`pickAlternatingVoices`)** — 앱에서 한 번만.
   - 조건: 해당 언어 + **오프라인(local)** + 뚜렷한 화자(`-x-` 포함) + **구형 htm 제외**.
   - `-local`/`-network` 중복은 베이스 이름으로 합쳐 서로 다른 2개만.
2. **문장 홀짝으로 적용 (`applyVoice`)** — 재생 루프에서 스텝마다.
   - `voices[sentenceIndex % 2]` → 문장 0,2,4…는 목소리 A / 1,3,5…는 목소리 B.
   - 목소리가 **바뀔 때만** `tts.voice`를 재설정(이음새 매끄럽게, 발화마다 재설정 금지 — 기존 원칙 유지).
   - `setVoice`가 언어도 같이 설정하므로 기존 언어 세팅과 동기화.

### 실제로 골라진 목소리 (갤럭시 S24+)
```
ja = [ja-jp-x-jab-local, ja-jp-x-jac-local]
ko = [ko-kr-x-ism-local, ko-kr-x-kob-local]
```
→ 일본어는 jab/jac, 한국어는 ism/kob를 문장마다 번갈아. 실기기 청취 결과 **남/여가 교대로** 잘 들림(확인 완료).

## 3. 삽질 메모 — 폰 USB 인증이 두 번 풀림

코드는 정상이었는데 `installDebug`가 두 번 실패했다. 원인은 **폰이 `adb`에서 `unauthorized` 상태**로 빠진 것(화면 잠김/케이블 재연결 등으로 USB 디버깅 승인이 풀림).
- 확인: `adb devices` → 기기가 `device`가 아니라 `unauthorized`로 뜸.
- 해결: 폰 **개발자 옵션 → USB 디버깅 승인 취소** 후 재연결 → 팝업에서 **항상 허용**. `device`로 뜨면 설치됨.
- 교훈: 빌드 실패 시 컴파일 에러부터 볼 게 아니라 `adb devices`로 **기기 상태**부터 확인.
